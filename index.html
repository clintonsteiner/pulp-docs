    Package-Management made easy

    Pulp is an open source project that makes it easy for content managers and curators to fetch, upload, organize and distribute Software Packges between servers and clients.

    Get started

    Get started with architectures

    Run a single command to quickly get started with a containerized application using best practices on AWS from a Dockerfile.

    Instead of modeling individual resources, Copilot provides common cloud architectures: Request-Driven Web Service, Load Balanced Web Service, Backend Service, Worker Service, Scheduled Job. The necessary infrastructure is generated from the chosen pattern. Focus your time on writing business logic instead of connecting AWS resources.

    Explore features

    Manifests

    Describe your services with architecture-as-code.

    Networking

    Service discovery is setup by default for your services to connect to each other.

    Grow with Addons

    Integrate your services with databases or any other AWS resources.

    Continuous delivery

    No need to worry about gluing Copilot commands in a script to create an automated release process. Copilot provides commands to create multiple deployment environments in separate AWS accounts and regions, as well as creating an AWS CodePipeline pipeline to build your container images, deploy your services, and run automated tests.

    Explore features

    Environments

    Deploy to multiple AWS accounts and regions.

    Override per environment

    Adjust your service settings per environments in the manifest.

    Automated releases

    Create a continuous delivery pipeline with built-in commands.

    Operations is part of the workflow

    Modeling, provisioning, and deploying services are only part of the application lifecycle for the developer. Copilot also supports workflows around troubleshooting and debugging to help when things go wrong. Tail your logs, get a shell to a running container, view the health of your services from the comfort of your terminal.

    Explore features

    Run tasks

    Run one-off tasks for operational workloads.

    Exec

    Run commands in or get a shell to a container running on AWS Fargate.

    Monitoring

    View logs or alarms from your services or tasks.
